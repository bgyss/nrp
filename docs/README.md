# Documentation

- [quickstart.md](quickstart.md) — every CLI, copy-pasteable: numpy/torch training,
  Mitsuba export, paper-scale runs, relighting, inverse lighting, benchmarking.
- [architecture.md](architecture.md) — how the code is organized: the path-cache data
  schema, the two backends, every CLI, and the training-config reference.
- [paper-mapping.md](paper-mapping.md) — the reimplementation guide: what each section
  of Sancho et al. (EGSR 2026) maps to in this repo, with every deviation and its
  rationale, and a flat known-deviations summary at the end.
- [performance.md](performance.md) — benchmark methodology and all measured results,
  with the hardware context needed to interpret them.
- [roadmap.md](roadmap.md) — the ten-item improvement roadmap, each written as a
  ready-to-run goal prompt with built-in verification and performance-testing
  requirements.
- [status/](status/) — dated status reports; each covers the items delivered since
  the previous report and gives a before/after picture of measured capability.
- [social/](social/) — external write-ups (e.g. LinkedIn posts) drafted from a
  status report, kept for reference.
