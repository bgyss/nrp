# Documentation

Start with [tracks.md](tracks.md) — the milestone progression that ties
everything below together.

## The tracks (in order)

- [tracks.md](tracks.md) — the spine: five phases, their status, and how they
  connect.
- [roadmap.md](roadmap.md) — phase 1 (complete): the ten-item replication
  roadmap, each written as a ready-to-run goal prompt with built-in
  verification and performance-testing requirements.
- [extensions.md](extensions.md) — phase 2 (complete): the E1–E9 extension
  program testing whether the decoupling is a building block for a real-time
  neural rendering pipeline.
- [pipeline-feasibility.md](pipeline-feasibility.md) — phase 3 (complete): the
  E10 verdict per target audience, with every claim traced to a report.
- [production-track.md](production-track.md) — phase 4 (complete): the
  trunk-and-branches ladder attacking the verdict's named blockers, ending in
  three summit demos — three rungs closed as honest negatives/partials.
- [hardening-track.md](hardening-track.md) — phase 5 (in progress): fixing what
  phase 4 surfaced — the QuadLight zero-collapse first — re-earning the
  caveated V1/V2 claims, and refreshing the verdict. Motivating audit:
  [status/2026-07-11.md](status/2026-07-11.md).

## Reference

- [quickstart.md](quickstart.md) — every CLI, copy-pasteable: numpy/torch training,
  Mitsuba export, paper-scale runs, relighting, inverse lighting, benchmarking.
- [architecture.md](architecture.md) — how the code is organized: the path-cache data
  schema, the two backends, every CLI, and the training-config reference.
- [paper-mapping.md](paper-mapping.md) — the reimplementation guide: what each section
  of Sancho et al. (EGSR 2026) maps to in this repo, with every deviation and its
  rationale, and a flat known-deviations summary at the end.
- [performance.md](performance.md) — benchmark methodology and all measured results,
  with the hardware context needed to interpret them.
- [status/](status/) — dated status reports; each covers the items delivered since
  the previous report and gives a before/after picture of measured capability.
- [social/](social/) — external write-ups (e.g. LinkedIn posts) drafted from a
  status report, kept for reference.
- [plans/](plans/) — dated design documents for major track decisions.
