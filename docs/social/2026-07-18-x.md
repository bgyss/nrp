## Post 1 — hook

Last week: a "perfect" 154.5 dB result that was half fake — 3 of 8 neural
light proxies were silently outputting exactly zero. This week: found out
why, and fixed it in one line. Thread on the debugging 🧵

## Post 2 — the setup

Each proxy: hashgrid encoding → MLP → softplus head, trained with the
paper's relative-MSE loss. 3 of 8 collapsed to exact-zero output during
training on the real scene. Same code trains fine elsewhere. Targets are
provably nonzero. So the bug is inside training.

## Post 3 — ruling things out

Ablated one variable at a time on the real failing case:
• ε 0.01→1.0: still collapses
• grad-norm clip @1.0: still collapses
• restricting light sampling to kill outlier-bright targets: still collapses
• hidden ReLU→LeakyReLU: still collapses
None of these were it.

## Post 4 — the lever

10× lower learning rate: no collapse. That's the actual lever, and it
pointed at the real mechanism.

## Post 5 — the mechanism

Output init predicts ~0.69; true target scale is ~130-700x dimmer. Adam's
step size is ~constant regardless of gradient size (ε/clip did nothing) —
so it walks pre-output toward -∞ steadily until softplus's derivative hits
exact 0. Dead, permanently. ~50/800 iters in.

## Post 6 — not quad-specific

Ran the identical setup on a working sphere light: same drift, just slower —
never quite crosses the dead zone in-budget. That's literally why it was
already the worst-performing "working" light before I knew any of this.

## Post 7 — the metric twist

Before the fix, the 3 broken proxies scored *higher* on validation PSNR than
the healthy light. Predicting zero is a decent guess against mostly-dark
validation targets. The metric didn't just miss the bug — it rewarded it.

## Post 8 — the fix + what's next

Fix: init the output bias to the true target scale instead of a generic
default. One method, 0.14s overhead, verified on the real scene — all 3 dead
proxies alive now, quality matches the working lights. Rung 1 of a
hardening pass on last week's claims. Repo in reply. 🔧
