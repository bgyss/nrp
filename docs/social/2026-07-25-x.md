## Post 5 — negative, flipped

The 2 texture-conditioned light proxies: more training time and a bigger
network do nothing (4x budget, 2x width — still 3-4dB short). Diagnosis:
their texture was flattened into 192 raw inputs, no structure. So I
restructured the model instead of the training.

## Post 5b — the fix

The emission is *linear* in the texture — each texel is an independent
emitter. New head: the MLP predicts a per-texel kernel, contracted with
the texture at the output. 19.64/19.99 dB — both inside the quality
envelope. At 1/4 the budget it beats every flat variant.

## Post 6 — honest negative

A toy-scale dynamic-geometry fix that looked promising 2 months ago does
NOT transfer to a real re-traced scene. Neither repair strategy meets the
1dB recovery target — closest misses by 14.5dB, the other by 40+dB. This
confirms the blocker is real, it doesn't soften it.

## Post 7 — what shipped clean

Shot storage: 1.17x raw frame size → 0.59x, just by keeping exact pixels
for a subset of "approval" frames only. Zero quality cost on any frame
tested.

## Post 8 — and

8-light rig compositing now runs on real WebGPU in real Chrome, ~1e-5
match vs CPU. First measurement: 170ms per slider nudge — a miss. But a
color nudge doesn't change the MLP inputs at all. Cache each light's raw
output, recomposite only. p95: 1.3ms. ~730x vs the CPU path.

## Post 8b — worst case too

Even the worst case — editing a light's *shape*, which invalidates one
light's cached contribution and re-runs its full MLP pass — lands at
30.9ms p95. Both scenarios beat the 33ms real-time stretch target. The
rig editor is now actually interactive.
