### Production-track results at a glance

**Trunk (shared infrastructure) — all four rungs met their targets.**

- **T1 — real-scene ingestion.** drjit wavefront exporter at 39–59× scalar
  throughput (target ≥ 20×); the Country Kitchen scene exported at 512²/64 spp
  and trained end to end (21.01 dB / SSIM 0.30 / ꟻLIP 0.146 at the initial
  3000-iter budget — later lifted to 29.4 dB by G2's 15k-iter retrain).
- **T2 — streaming discipline.** Streamed training peaks at 0.80 GiB resident
  vs 8.45 GiB monolithic on the packed kitchen cache, with 0.000 dB parity;
  packed-cache fidelity floor 44.90 dB; export memory ceiling documented.
- **T3 — perceptual gates.** `nrp.quality.gate` with preview/draft/final
  tiers; 4.3% overhead on the approval flow (< 5% target); a deliberately
  degraded render demonstrably fails; 19 unit tests.
- **T4 — runtime baseline lock.** The real 409k-param kitchen proxy
  (hashgrid ported to WGSL) in real Chrome: 1.2e-6 parity vs PyTorch, 512²
  p95 23.3 ms (43 fps p95) — the 30 fps floor holds with ~30% headroom;
  `mise run t4-check` is a committed regression gate. Getting there took a
  ~10× shader optimization (unrolled register accumulators + workgroup weight
  tiles); the failed variants are documented.

**Games branch.**

- **G1 — dynamic geometry, second attempt (honest negative, changed failure
  mode).** The frozen-base + invalidated-shard residual proxy reaches
  37.15 dB vs E2's exact 44.88 dB recovery reference — still 7.7 dB short of
  the 1 dB target, so the letter of the target remains unmet. But the failure
  mode is categorically different, which is what the rung asked to establish:
  out-of-mask fidelity is 54.90 dB (vs E2's 22.95 dB forgetting) and cannot
  drift by construction; the per-frame floor is 25.3 dB (vs E2's 10.6 dB
  collapse); at the per-frame median the residual is 1.17 dB *better* than
  full retrain, at 0.60× the wall-clock. The remaining gap is concentrated in
  two near-static frames where full retrain overfits to 110+ dB.
- **G2 — summit: live relight + controls in the browser (met).** 481 frames
  under real interaction at 512²: p95 30.7 ms ≤ 33 ms; 12/12 sampled trace
  frames pass preview tier vs OIDN-denoised GATHERLIGHT (SSIM ≥ 0.883);
  G1-panel GPU parity 4.8e-7; committed recording + interaction trace.
  Required retraining the T1 proxy (15k iters, cosine schedule → 29.4 dB).

**Film branch — both rungs met, one storage negative.**

- **F1 — shot harness.** 120/120 frames pass preview tier (31.6–37.8 dB);
  the new temporal FLIP-delta metric passes at worst excess 0.0019 ≤ 0.02
  while a 30 dB/frame flickering baseline fails at 0.165 — per-frame PSNR
  provably cannot see this failure mode.
- **F2 — summit: final-tier shot.** 120/120 frames pass the final-tier gate
  (≥ 105.9 dB) via fp16-stored residual-identity reconstruction; 118×
  wall-clock amortization vs re-exporting every frame; committed MP4.
  **Honest negative:** proxy + residual storage is 1.17× the raw frames —
  residuals at this proxy quality are noise-dominated and compress worse than
  the frames themselves. The win is time, not bytes.

**VFX branch — both rungs done, both substantially caveated (see negative
findings).**

- **V1 — production light rig (honest negative).** 8-light rig (3 sphere +
  3 quad + 2 textured-quad), one independent 800-iter proxy per light.
  Additivity gate at preview tier: PSNR 25.6 dB passes; SSIM 0.622 and FLIP
  0.352 both fail (thresholds 0.80 / 0.15). Per-light rig is 24.1× the size
  of a matched-budget monolithic baseline (the cost of relightability);
  compositing is linear at ≈ 111 ms/light at 512² on CPU torch.
- **V2 — summit: art-direction loop (partial recovery).** `optimize_colors`
  drives the composited rig to a hand-authored target: image-space draft-tier
  gate passes at 154.5 dB, the recovered rig round-trips bit-identically, and
  the headless slider loop measures 950.1 ms mean per adjustment. But only
  3 of the 6 colorable lights (key/fill/rim — all spheres) were genuinely
  gradient-recovered; the 3 quads had zero-gradient proxies inherited from V1
  and their "recovery" is the untouched neutral guess.

## Negative Findings and Open Issues (ranked)

1. **QuadLight proxies collapse to exactly-zero output on the kitchen cache —
   root cause narrowed by this audit, not yet fixed.** V2's report showed
   `window`/`ceiling_panel`/`practical` (all `QuadLight`) produce raw proxy
   output with `mean: 0.0, max: 0.0`, leaving only 5 of the "8-light" rig
   genuinely contributing and 3 of 6 V2 recoveries vacuous. The V1/V2 docs
   established what it is *not* (training budget: `rim` has the worst val
   PSNR of the rig and works) but left the cause undiagnosed. **New probe
   (this audit):** running numpy `gather_throughput_quad` on
   `out/kitchen-512/path_cache.npz` at the three authored quad params gives
   distinctly **nonzero** GATHERLIGHT targets — mean 0.0028–0.0053, max
   ≈ 1.04, 13–30% of pixels lit — the same order as `rim`'s sphere target
   (mean 0.0082, 27% lit), which trained to a working proxy. So the failure
   is **training-side, not target-side**: the supervision signal exists at
   the authored params, and the proxy still lands in softplus's exact-zero
   regime. Two facts bound the search: quad proxies trained fine on the toy
   and Cornell caches (`out/toy-quad-torch`, `out/mitsuba-quad-torch`,
   roadmap item 4), so it is cache × type specific; and §4.4 pool training
   samples quad *normals uniformly on the unit sphere*
   (`nrp/torch_backend/sampling.py`) while positions come from recorded
   segments — so most sampled pool targets for quads on this cache are
   plausibly near-black, and Eq. 4's relative MSE (stop-grad denominator,
   ε = 0.01) has a well-known zero-collapse basin: once predictions go dark,
   gradients vanish. That is the leading hypothesis, not a conclusion —
   diagnosing it is rung H1 of the next phase.
2. **V1 additivity fails preview tier on SSIM/FLIP** (0.622 / 0.352). Cannot
   be interpreted until (1) is fixed — 3 of 8 layers are missing from the
   composite. Re-run after the quad fix; if it still fails, the
   textured-quad proxies are the next suspect.
3. **Textured-quad proxies are the genuine low-quality contributors**
   (12.35 / 13.35 dB val PSNR, 3.4× slower to train). Harder,
   higher-frequency targets at the same 800-iter budget; G2 showed the same
   architecture reaches 29.4 dB at 15k iters on the full scene, so budget
   scaling is the first lever.
4. **Interactive grading latency is ~29× off real time** (950 ms mean per
   slider adjustment; 8-light CPU-torch compositing at ≈ 111 ms/light). T4
   proved the same proxy at 21 ms/frame in WebGPU — the rig has simply never
   been ported to the proven GPU runtime.
5. **V2's optimizer wall-clock (1320.7 s) carries a known avoidable
   inefficiency**: the 2 textured-quad lights get a full forward pass on all
   500 steps despite contributing a constant — ~2/8 of per-step compute.
6. **F2 storage negative**: residuals cost 1.17× raw frames at fp16. Needs
   residual quantization/subsetting (approval-frames-only) to flip.
7. **G1's real-scene feasibility is untested** — the Mitsuba exporter has no
   scene-edit/retrace path, so residual recovery is proven only at toy scale
   (and shipped as a labeled toy panel in G2).
8. **Preview-tier gating is slower than preview rendering** (gate 0.146 s vs
   proxy render 0.120 s at 512²) — fine for approval flows, unusable
   per-frame in interactive contexts; T3 flagged this for the runtime path.
9. **Verified-region caveat propagates**: G2/F1 light paths were authored
   inside the proxy's verified pass region; arbitrary light positions do not
   gate at preview tier. This is a distribution-coverage limitation of the
   trained proxies, worth an explicit rung if the demos are to generalize.
10. **The E10 verdict document predates the production track.** Its verdict
    rows don't yet reflect T1–V2 evidence (e.g. "scale proof is
    cornell-box-only" is now false). Needs a refresh pass.

## Plan Going Forward — Phase 5

The production track answered its question: every named E10 blocker now has a
measured result, and the summits shipped. What it *surfaced* is a different
class of work — the pipeline's first genuinely undiagnosed model-behavior bug
(quad zero-collapse), plus a set of quantified gaps between "measured" and
"usable" (rig latency, storage, coverage). The next phase is a **hardening
track**: fix the diagnosed failures, re-earn the two caveated summit claims,
and refresh the verdict. Full rung-by-rung goal prompts live in
`docs/hardening-track.md`; summary:

| rung | goal | exit criterion |
|---|---|---|
| H1 | Diagnose and fix the QuadLight zero-collapse | a kitchen-cache quad proxy with nonzero output and val PSNR parity with spheres; the mechanism written up (whether sampling, loss basin, or otherwise) |
| H2 | Re-earn V1/V2 honestly | additivity re-gated with 8/8 contributing proxies at a budget informed by G2's 15k-iter result; V2 re-run targeting 6/6 genuine recoveries; the known optimizer inefficiency fixed |
| H3 | Textured-quad quality | budget/capacity sweep to bring textured quads within the rig's quality envelope, or a documented finding that texture conditioning needs a different input scheme |
| H4 | Rig on the proven WebGPU runtime | port N-light compositing to the T4/G2 shader path; slider-loop latency target ≤ 100 ms p95 at 512² for the 8-light rig |
| H5 | Real-scene dynamic geometry | exporter scene-edit/retrace support so G1's residual recovery can be evaluated on the kitchen, not just the toy box |
| H6 | Flip the F2 storage negative | residual quantization sweep + approval-frame subsetting; storage < 1× raw frames at unchanged gate verdicts, or a documented floor |
| H7 | Verdict refresh | update `docs/pipeline-feasibility.md` verdicts against T1–V2 + H1–H6 evidence |

Ordering: H1 blocks H2; H2 partially blocks H4 (port the fixed rig, not the
broken one); H3, H5, H6 are independent; H7 last. Same conventions as every
phase: JSON report per claim, `docs/performance.md` entry with hardware
context, honest negatives are deliverables.
